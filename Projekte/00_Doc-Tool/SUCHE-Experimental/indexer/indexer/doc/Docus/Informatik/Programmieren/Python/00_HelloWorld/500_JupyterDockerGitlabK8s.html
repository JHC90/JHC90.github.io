JUPYTER on Docker executed on Gitlab & K8s
am Beispiel MNIST

Dieses Notebook dient dazu, die Funktionsweis von Jupyter im Zusammenhang mit Docker github und K8s zu verstehen. Konkret versuche ich das Notebook lokal auf meinem Development-Rechner zu entwicklen. Dieses Notebook soll später auf dem K8s Cluster ausgeführt werden.

Somit gibt es zwei mögliche Ausführungsorte. Innerhalb einer Konditionalverzweigung erkennt Python ob es gegenwärtig auf meinem lokalen Rechner oder eben nicht auf meinem lokalen Rechner ausgeführt wird. Je nach dem Ausführungsort wird entweder eine nicht komplexe Ausführung getriggert oder eben eine komplexe Ausführung. Die Komplexität unterscheidet sich darin, wieviele Algorithmen (in dem vorliegenden MNIST-Beispiel Klassifikationsalgorihtmen) auf den Datensatz angewendet werden.

Hier wird das MNIST-Problem abgearbeitet, dabei habe ich eine umfangreiche "Ohrfeigenfunktion" hinterlegt. Konkret heist dass das auf meinem Development Rechner die Gesamte Struktur etwa 40 Sekunden an der "nicht-komplexen" und etwa 5h an der komplexen Konfiguration rechnet.

Ziel des Notebooks ist es das Rechnen grundästzlich an den K8s Cluster auszulagern.

Was ich mit diesem Notebook herausfinden möchte ist, wie das fertig berechnete Notebook mir anschließend wieder zurückgeliefert wird. Meine gegenwärtige Idealvorstellung wäre - Erstellen und warten des Notebooks grunsätzlich lokal auf dem Development Notebook - Ausführung in der nicht komplexen Config auf dem Development Notebook => proofe dass die Entwicklung funktioniert - Push auf git - K8s Startet mit Yml automatisch - Das Notebook wird mit der komplexen Konfiguration ausgeführt - Das fertig trainierte Notebook wird als Artifakt im GIT-Repo hinterlegt - diese Artefakt kann manuell heruntergeladen werden - Lets see!

Das lokale Repo ist mit einem Remote repo Verbunden - öffentliches gitlab / github - Gitlab der Fh-Kufstein - ... 3. der öffentliche Git Server ist mit einem K8s Cluster verbunden 4. Ohrfeigenfunktion für Klassifikationsalgorithmen

Die Ohrfeigenfunktion ist eigenlich "nur eine Schleife". Wird diese jedoch geschickt eingesetzt können somit eine Anzahl von Klassifikationsalgorithmen automatisch ausgeführt, und die jeweiligen Ergebnisse in einer Rückgabeliste gespeichert werden. Diese Rückgabeliste mit den Ergebnissen kann später mit einer entsprechenden Grafik ausgewertet und verglichen werden.
